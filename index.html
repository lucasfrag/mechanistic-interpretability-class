3
O ponto de partida

Por dentro, o modelo é só números

Um LLM funciona, mas por dentro é uma caixa-preta: bilhões de pesos que produzem, a cada camada, um vetor de ativações — listas de números sem rótulo.

A aposta da interpretabilidade é que esses números não são aleatórios: eles codificam conceitos — e a primeira pergunta é onde cada conceito se esconde ali dentro.

Achar isso separa "o modelo acertou" de "eu sei por quê" — a base de tudo que vem depois.

arquitetura → Attention is All You Need (Vaswani et al., 2017)

Representação · features

O significado está na direção

A primeira descoberta:

aqueles números têm estrutura. O espaço de ativação tem d dimensões (os neurônios), mas um conceito raramente mora num neurônio só — ele é uma direção que combina vários.

dimensão = capacidade do espaço · direção (uma feature) = como um conceito se aloja nele.

Por isso um neurônio isolado quase nunca é um conceito limpo.

E se conceitos são direções, combinar conceitos vira aritmética de vetores.

até aqui sabíamosativações são vetores de números

Representação · direções se combinam

Aritmética de conceitos

Cada conceito é uma direção — um vetor a partir da origem. E se conceitos são vetores, combiná-los vira soma e subtração. Partindo de rei, subtraio a direção homem, somo mulher, e chego perto de rainha.

O preço é a interferência

Duas features não perpendiculares “vazam” uma na outra. Essa medida de vazamento é o produto interno entre suas direções.

Superposição · a consequência

Um neurônio ≠ uma feature

Se as features se sobrepõem, um neurônio isolado responde a várias features não relacionadas ao mesmo tempo.

  • Isso é polissemanticidade — olhar um neurônio não revela um conceito limpo.
  • A feature verdadeira está espalhada por muitos neurônios, e cada neurônio mistura muitas features.

Então, como desembaralhar as features de volta?

SAE · como lemos uma feature

Uma feature não vem rotulada

O ponto honesto:

o SAE separa as features, mas cada uma é só um vetor com um número — ele não diz "isto é a Golden Gate". A gente descobre o significado por fora.

Como? E de onde vêm os exemplos?

rodamos um corpus grande (o mesmo tipo de texto em que o modelo treinou) pelo modelo, guardamos quanto a feature dispara em cada trecho, e pegamos os que mais a ativam. Se todos falam da mesma coisa, esse é o conceito dela.

É leitura indireta: não lemos o vetor, lemos os exemplos que ele destaca — e batizamos a feature com o padrão que aparece.

até aqui sabíamoso SAE dá features esparsas e separadas
mas falta um passoa feature é um vetor mudo — o rótulo vem dos top-exemplos

SAE · rotular em escala

Um modelo para explicar o outro

O problema de escala:

um LLM de produção tem milhões de features — ninguém lê os top-exemplos de cada uma à mão.

A saída:

mostrar os top-exemplos para outro LLM e pedir que ele proponha o rótulo.

Circuitos · o que é

Um caminho de cálculo

Já sabemos onde…

as features vivem. Mas features soltas não explicam o cálculo — como o modelo combina peças para resolver uma tarefa?

A ideia:

componentes — cabeças de atenção, neurônios de MLP — lêem features de camadas anteriores e escrevem novas adiante, ligados numa cadeia.

Circuitos · indução

“Já vi isso antes — copio o que veio depois”

Um circuito concreto:

ao ver “Harry” pela 2ª vez, o modelo prevê “Potter” — porque foi o que seguiu o 1º “Harry”.

Duas cabeças em sequência bastam:

  • previous-token: marca quem veio antes de cada token.
  • induction: volta à ocorrência anterior e copia o próximo.

Circuitos · da observação à prova

Observar não é provar

Traçamos o circuito, mas…

ver duas cabeças "acenderem juntas" é só correlação. Elas realmente causam a previsão — ou só acompanham?

A prova de verdade: mexer no circuito e ver a saída mudar. Se desligo a induction head e a previsão quebra, ela era causal.

Intervenção · attribution patching
técnicaattribution patching

De onde vêm os gradientes?

A pergunta natural:

se não vamos rodar mil forward passes, o que mede o efeito de cada componente? O gradiente.

Três passos, dois passes:

1) um forward guarda as ativações. 2) um backward propaga a métrica de volta — é daí que saem os gradientes ∂m/∂a, um por componente de uma vez.

3) aproximação linear (Taylor 1ª ordem): efeito ≈ grad × (corrompido − limpo).

Attribution Patching (Nanda 2023)

Circuitos · descoberta automática
técnicaACDC · critério

Cortar só o que não faz falta

A pergunta para cada aresta:

"se eu cortar esta conexão, a saída do modelo muda muito?" A resposta decide se ela fica ou sai.

Como medir a mudança:

a divergência KL é um número que mede o quanto duas distribuições de probabilidade diferem — aqui, a saída com vs sem a aresta. 0 = idênticas; quanto maior, mais a aresta importava.

τ (tau) é um limiar que a gente escolhe: se a KL fica abaixo dele, a aresta é irrelevante → corta; senão, mantém.

τ menor = circuito maior e mais fiel. Towards Automated Circuit Discovery (Conmy 2023)

Intervenção · steering via SAE
técnicaSAE steering · mecânica

De onde vem a direção?

O SAE tem duas metades:

o encoder lê a ativação h e diz quais features estão ativas (um vetor esparso — quase tudo zero). O decoder reconstrói h a partir delas.

repara: o decoder é uma soma de vetores. Cada feature i tem o seu vetor-coluna di — e a ativação dela diz o quanto usar dele.

Esse vetor-coluna di é a "direção da feature". Não precisamos procurá-la: o próprio SAE já a fornece, pronta para o steering.

até aqui sabíamoso SAE dá features interpretáveis
a mecânicaa coluna do decoder de cada feature é a sua direção dSparse Autoencoders Find Highly Interpretable Features (Cunningham 2023) · Scaling Monosemanticity (Templeton 2024)
