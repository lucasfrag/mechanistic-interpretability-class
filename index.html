Bloco 1 · features

O significado está na direção

O espaço de ativação tem d dimensões — os eixos, os neurônios. Uma feature não é um eixo: é uma direção qualquer, combinando várias dimensões.

dimensão = capacidade do espaço · direção = como a feature se aloja nele.

Por isso um neurônio isolado quase nunca é um conceito limpo.

E se conceitos são direções, combinar conceitos vira aritmética de vetores.

Features viram formas geométricas

Para minimizar interferência, N features espalham-se o mais longe possível umas das outras — nos vértices de um polígono regular.

—

2 features → linha (antípodas, 180°)
4 features → quadrado (90°, ortogonais!)
5+ features → pentágono, hexágono… (< 90°, interferência)

O preço é a interferência

Duas features não perpendiculares “vazam” uma na outra. Essa medida de vazamento é o produto interno entre suas direções.

cos 90° = 0 → ortogonais, interferência zero (ideal)
cos 0° = 1 → mesma direção, interferência máxima (evitada)

Bloco 2 · a consequência

Um neurônio ≠ uma feature

Se as features se sobrepõem, um neurônio isolado responde a várias features não relacionadas ao mesmo tempo.

  • Isso é polissemanticidade — olhar um neurônio não revela um conceito limpo.
  • A feature verdadeira está espalhada por muitos neurônios, e cada neurônio mistura muitas features.

Então, como desembaralhar as features de volta?

Bloco 3 · o que ganhamos (e o que falta)

Features que a gente lê

Depois de treinar o SAE, olhamos o que faz cada feature disparar: quais trechos de texto a acendem. Cada uma acaba correspondendo a um conceito reconhecível.

Ex.: uma feature só acende quando o texto fala da ponte Golden Gate; outra, só em código Python. Viram detectores de conceito que a gente lê e nomeia — um por um.

  • Em escala: milhões deles num LLM de produção.
  • Mas com ressalvas: muitas features nunca disparam (aprendidas à toa); o x̂ reconstruído nunca é idêntico ao original; e dizer se uma feature é mesmo "um conceito só" ainda depende de um humano olhando exemplos.
até aqui sabíamosesparsidade dá features interpretáveis
a demonstraçãofeatures nítidas em escala real (ex.: Golden Gate)Templeton et al. 2024 · Scaling Monosemanticity
novo desafio→ features soltas não explicam o cálculo. Circuitos
Bloco 4

Circuitos: features
que computam juntas

Uma feature sozinha é um substantivo. O verbo aparece quando várias se conectam entre camadas para executar um algoritmo.

16
Bloco 4 · a ideia

Um caminho de cálculo

Componentes do modelo — cabeças de atenção, neurônios de MLP — lêem features de camadas anteriores e escrevem novas adiante.

Um circuito é o subconjunto desses componentes que, juntos, implementam uma tarefa específica — ligados numa cadeia de causa e efeito.

Achar o circuito = achar quais peças importam para aquela tarefa, e como a informação flui entre elas.

até aqui sabíamosfeatures são os "substantivos" do modelo
a ideiaconectadas, formam circuitos que rodam algoritmosOlah et al. 2020 · Zoom In
exemploo circuito de indução, passo a passo

Bloco 4 · o circuito de indução

“Já vi isso antes — copio o que veio depois”

Ao ver “Harry” pela 2ª vez, o modelo prevê “Potter” — porque foi o que seguiu o 1º “Harry”.

Duas cabeças em sequência bastam:

  • previous-token: marca quem veio antes de cada token.
  • induction: volta à ocorrência anterior e copia o próximo.

Bloco 4 · da observação à prova

Observar não é provar

Ver duas cabeças "acenderem juntas" é correlação. Elas realmente causam a previsão — ou só acompanham?

A prova de verdade: mexer no circuito e ver a saída mudar. Se desligo a induction head e a previsão quebra, ela era causal.

É aí que a interpretabilidade deixa de descrever e passa a testar.
